No simulation! Everything should be as real as possible!

Yeah! You've learnt previously how to simulate a screen resolution. But the problem here, that it's just simulationng. You never know how a web page behaves on the real device.

Here is the tough task! Sometimes, you have no remote server to test a web app and you're forced to run a dev site on the localhost. Read this article Access Local Server. Now, load the archive from here Archive , make sure you have node.js installed on your machine. Extract an archive and go to the frontent directory. From this directory run commands in your command line.

npm i
npm start

After the server starts, go to localhost:4200. Now, access this localhost on your Android device. It's not hard, just select "Port forwarding" and "Add Rule" "4200: localhost:4200" as on the image below.


Now you can start you testing even before the real backend would be deployed!
