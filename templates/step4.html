2) Is it any bug message in the response? Let's create a bug report (Google Docs is enough for a bug)! Write down steps to reproduce a bug and then copy a request as cURL format into the bug report

3) Now, right click on any request in the Name section and select "save as HAR with content".

"F12" key --> 'Network" tab --> right click on "getjson" --> "Save as HAR with content"


Go to this website and load a har file.
What is HAR and why we need this? HAR is basically a copy of all your requests. When you report a bug, sometimes, when it's unclear what API call caused a bug, it's better to provide to a developer a har file, so the developer can analyze all you network activity.

4) Let's find out how our site works with slow internet. Open a "Network conditions" tab and select 'Slow 3G".

"F12" key --> ⋮ button --> ""More tools" --> 'Network conditions"

Make sure "Capture screenshots" is enabled. Click 'F5" and wait until the page has been loaded.
If everything has been done correctly, you can see a bunch of screenshots in the Network tab. Double click on it and inspect how our wep page has been loaded under slow 3G.
What do you think? Why it's useful to test under slow 3G connection? Explain to your mentor.
